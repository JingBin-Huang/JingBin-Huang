低代码平台的回撤功能实现

前言​

​ 回撤功能是低代码平台中最常见的功能，也就是在电视剧中出现过的后悔药，棋类游戏中的悔棋操作等。低代码平台中的回撤功能能够回撤到上一步以及更多步的配置。在日常生活中，与之相像的就是浏览器的历史记录，每个标签页面都存在前进和后退的功能。接下来详细说说回撤功能的实现方式。

初步思考​

​ 历史记录，其实就是一串有序的数据。

​ 首先需要一个容器将每一步的数据进行记录，例如：Array（数组）的方式进行存储数据。每一个新的操作都可以将新的数据加入push到这个容器Array中。

​ 其次，我们需要记录每一次的最新记录的下标，初始值为 -1，每次加入push到这个容器Array中时，下标+1。这样，下标对应的其实就是Array下标中对应的每一步的数据。

​ 考虑边界条件，在回撤过程中，每一个回撤记录的下标递减，什么情况下不能回撤，根据上面的描述，我们可以知道当下标为-1的情况时，Array中没有与之相对应的数据的时候，当前页面已经回到最初始的状态。而在前进操作中，每一个回撤记录的下标递增，当下标在Array中没有与之相对应的数据的时候，当前页面已经是最新记录的状态。

实现​

  1. 先建一个HistoryClass 类，根据上面的描述，存在两个成员变量，记录历史的容器，和当前记录的下标。同时存在三个成员方法，添加历史记录，回撤以及前进功能。
class HistoryClass {
        list = [] // 存储历史记录
    index = -1 // 当前记录的下标
  
        addHistory(data) {
        this.list.push(data)
        this.index++
    }
  
    prevHistory() {
        return new Promise((resolve, reject) => {
            // 发现下标为 -1 或者 下标减1等于 -1 的情况， 即不可回撤
            if (this.index === -1 || --this.index === -1) {
                return reject()
            }
  
            const res = this.list[this.index]
            resolve(res)
        })
    }
  
    nextHistory() {
        return new Promise((resolve, reject) => {
            // 当下标为最新时，不可前进
            if (this.index + 1 === this.list.length) {
                return reject()
            }
            this.index++
            const res = this.list[this.index]
            resolve(res)
        })
    }
}

export default HistoryClass
  1. 再次进行思考🤔️，正常在与UI界面进行交互时，需要在不可回撤或者不可前进时，将按钮进行禁用，避免让用户觉得这个功能是一直都是有效果的。可以添加两个状态来描述当前是否能够回撤或者前进，同时在每次前进或者回撤时更新状态。需要注意⚠️，更新状态的时候，我们需要的是预测下一步是否能够回撤或者前进，否则将毫无意义。
class HistoryClass {
        list = [] // 存储历史记录
    index = -1 // 当前记录的下标
        canPrev = false // 可回撤
      canNext = false // 可前进
  
        addHistory(data) {
        this.list.push(data)
        this.index++
    }
  
    prevHistory() {
        return new Promise((resolve, reject) => {
            // 发现下标为 -1 或者 下标减1等于 -1 的情况， 即不可回撤
            if (this.index === -1 || --this.index === -1) {
                return reject()
            }
          
            updateStatus(this.index - 1 !== -1, true)
            const res = this.list[this.index]
            resolve(res)
        })
    }
  
    nextHistory() {
        return new Promise((resolve, reject) => {
            // 当下标为最新时，不可前进
            if (this.index + 1 === this.list.length) {
                return reject()
            }
            this.index++
            updateStatus(true, this.index + 1 !== this.list.length)
            const res = this.list[this.index]
            resolve(res)
        })
    }

        updateStatus(canPrev, canNext) {
        this.canPrev = canPrev
        this.canNext = canNext
    }
}

export default HistoryClass
  1. 这个时候还需要考虑，怎么及时通知使用者这个按钮是可用不可用呢？这个时候就需要知道怎么主动将更新状态的消息推送给用户。可以注册一个更新按钮状态的函数，在每次更新状态的时候将canPrev和canNext当作函数参数，进行调用。
class HistoryClass {
        list = [] // 存储历史记录
    index = -1 // 当前记录的下标
        canPrev = false // 可回撤
      canNext = false // 可前进
        updateStatusCallback = null // 更新状态的回调函数
  
        addHistory(data) {
        this.list.push(data)
        this.index++
    }
  
    prevHistory() {
        return new Promise((resolve, reject) => {
            // 发现下标为 -1 或者 下标减1等于 -1 的情况， 即不可回撤
            if (this.index === -1 || --this.index === -1) {
                return reject()
            }
          
            updateStatus(this.index - 1 !== -1, true)
            const res = this.list[this.index]
            resolve(res)
        })
    }
  
    nextHistory() {
        return new Promise((resolve, reject) => {
            // 当下标为最新时，不可前进
            if (this.index + 1 === this.list.length) {
                return reject()
            }
            this.index++
            updateStatus(true, this.index + 1 !== this.list.length)
            const res = this.list[this.index]
            resolve(res)
        })
    }

        updateStatus(canPrev, canNext) {
        this.canPrev = canPrev
        this.canNext = canNext
        this.updateStatusCallback && this.updateStatusCallback(canPrev, canNext)
    }

        registerUpdateCallback (func) {
      this.updateStatusCallback = func
    }
}

export default HistoryClass

再次思考，另一种实现方式​

​ 可以使用两个容器记录可回撤的数据记录Prev，和一个可前进的数据记录Next，还有一个当前的记录。每回撤一步，先将当前记录放入Next中，然后从Prev中的最上层获取一份数据，当做新的当前记录。前进一步与之相反。

实现​

class HistoryClass {
    current = null
        prevList = []
        nextList = []
        canPrev = false // 可回撤
      canNext = false // 可前进
        updateStatusCallback = null // 更新状态的回调函数

        addHistory(data) {
        if (this.current) {
            this.prevList.push(data)
        }
        this.current = data
    }

        prevHistory() {
        return new Promise((resolve, reject) => {
            const temp = this.prevList.pop()
            if (temp) {
                this.nextList.push(this.current)
                this.current = temp
                this.updateStatus()
                return resolve(this.current)
            } else {
                reject()
            }
        })
    }

        nextHistory() {
        return new Promise((resolve, reject) => {
            const temp = this.nextList.pop()
            if (temp) {
                this.prevList.push(this.current)
                this.current = temp
                this.updateStatus()
                return resolve(this.current)
            } else {
                reject()
            }
        })
    }

        updateStatus() {
        this.canPrev = this.prevList.length
        this.canNext = this.nextList.length
        this.updateStatusCallback && this.updateStatusCallback(this.canPrev, this.canNext)
    }

        registerUpdateCallback (func) {
      this.updateStatusCallback = func
    }
}

相比第一种实现方式，这种方式更简单，更易理解。应用与第一种方法一致。

总结​

​ 每种方式都有自己的优点和缺点，根据使用场景选择最合适的才能算是最优的方案。
